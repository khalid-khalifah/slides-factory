## What Was Built

### `_deep_merge(base, override)` тАФ private helper
Recursive dict merge with list-replace semantics: child lists replace parent lists (never append). Child scalars override parent scalars. Nested dicts merge recursively.

### `_load_brand_raw(path, _chain=None)` тАФ private resolver
Reads a brand YAML, resolves `extends` inheritance, and returns the fully merged raw dict (before `BrandTheme` construction).

Key behaviours:
- `extends` accepts a **single string** or a **list of paths** (ordered merge)
- Later `extends` entries override earlier ones; the child's own values override everything
- Resolution is relative to the child's parent directory (`path.parent / rel_path`)
- Circular chains detected via a private `_chain` parameter тЖТ raises `SlidesFactoryError`
- Chains deeper than 5 levels тЖТ `logger.warning()`
- Missing parent files тЖТ standard `FileNotFoundError`

### `load_brand(path)` тАФ updated
Now delegates to `_load_brand_raw()` for the merge, then constructs `BrandTheme` from the merged dict. Backward compatible: YAML files without `extends` load identically to before.

---

## API Surface

No new public API. The `extends` key appears only in brand YAML files. `load_brand()` retains its existing signature.

### Usage

```yaml
# brand.yaml  (child)
extends: base.yaml
name: "Acme Brand"
colors:
  main:
    - color: "#00A3E0"
      contrast: "#FFFFFF"
```

```yaml
# brand.yaml  (multi-inheritance, ordered merge)
extends:
  - slides-factory-basic.yaml
  - acme-colors.yaml
colors:
  main:
    - color: "#FF6600"   # child wins over both parents
      contrast: "#FFFFFF"
```

### Merge Rules

| Key | Strategy |
|-----|----------|
| `extends` | Consumed during resolution, excluded from merged output |
| `colors.main/secondary/basic` | Child **replaces** parent list entirely |
| `fonts.slots` | Child overrides by slot key |
| `layout.elements` | Shallow merge тАФ child overrides by element key |
| `layout.logos` | Deep-merge тАФ child locale entries override parent |
| `page` | Child **replaces** parent entirely (field-level merge not attempted) |
| `logos` | Child overrides by logo key |
| `name`, `default_frame`, `base_pptx`, `lock_frame_shapes` | Child wins if present |

# v0.4 тАФ Feature 4.3: Layout Debug Mode

**Status:** тЬЕ Complete тАФ Ready for Chief Programmer sign-off

**Branch:** `v0.4-debug-mode`

---

## What Was Built

### `slides_factory/layout/debug.py` тАФ `render_debug_layer()`

A dedicated debug module that renders diagnostic shapes **behind** the normal
slide content (python-pptx creates shapes in order, so debug shapes come
first and content appears on top).

**Rendered diagnostics:**

| Layer | What | Colour | Look |
|-------|------|--------|------|
| Playground boundary | Dashed red rectangle | `#FF0000` | 1pt dash |
| Padding region | Semi-transparent gold fill at playground edges | `#FFD700` | 15% alpha |
| Grid lines | Dashed gray column/row track boundaries | `#888888` | 0.25pt dash |
| Gap regions | Light fill bands between tracks | `#DDDDDD` | 12% alpha |
| Cell boundaries | Thin solid blue per-cell rectangle | `#3366FF` | 0.25pt solid |
| Cell labels | `[row,col]` text in top-left corner | Black | 8pt |

### Integration тАФ `ctx.debug` flag on `RenderContext`

Rather than threading a `debug` parameter through every function in the call
stack, the flag is stored on `RenderContext.debug` (a frozen dataclass field).
`render_layout()` checks `ctx.debug` and calls `render_debug_layer()` before
rendering content. All intermediates тАФ `RenderContext.with_debug()`, engine
`prepare_render(debug=True)`, document API тАФ exist to propagate the flag from
the CLI to the context.

### CLI commands updated with `--debug`

| Command | `--debug` added |
|---------|----------------|
| `slide new` | тЬЕ тАФ renders grid frame, lines, labels |
| `slide add` | тЬЕ тАФ renders behind template content |
| `slide add-many` | тЬЕ тАФ applied to every rendered slide |

### Python API

```python
# All three document functions accept debug=True
result = document.new_grid_slide(prs, app=app, grid="grid-cols-2 grid-rows-2", debug=True)
result = document.add_slide(prs, "simple", data, app=app, debug=True)
results = document.add_slides_from_rows(prs, "simple", rows, app=app, debug=True)
```
