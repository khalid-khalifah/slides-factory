### Feature 4 — Rich Text / Inline Formatting ✅

**Date:** 2026-06-26

**What was built:**

A new document-model-based rich text system that replaces the old flat `text`+`bullets` API with a composable tree structure, plus a complete pipeline to render it into python-pptx.

**New package:** `slides_factory/elements/text/` — the rich text core:
- `model.py` — `TextRun`, `Paragraph`, `ListItem`, `ListStyle`, `TextBlock`
- `dsl.py` — `text()` DSL constructor
- `html.py` — `parse_html()` with inline tags + list support
- `render.py` — `prepare()` transforms `TextBlock` → render-ready paragraphs

**Rewritten:** `slides_factory/elements/text.py`:
- `TextProps` now has `block: TextBlock` (with default empty block) — full rich text model
- `render_text()` uses `prepare()` to resolve the tree then draws each run with per-run `font.bold`, `font.italic`, `font.color.rgb`, `font.size`, `hyperlink.address`, etc.
- No style field in props — style comes through the element's `style_model` (clean separation)

**Updated:** `slides_factory/cli.py`:
- `_build_model_data()` now auto-parses JSON values (values starting with `{` or `[`)
- Enables `--set block='{"children": [{"runs": [{"text": "Hi"}]}]}'`

**Updated:** `slides_factory/document.py`, `slides_factory/templating.py`:
- All `model_dump(mode="json")` calls now pass `exclude_none=True` to produce clean output
